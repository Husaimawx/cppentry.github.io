<p>如下中的stackoverflow的链接。</p>
<p>简单的说，-put更宽松，可以把本地或者HDFS上的文件拷贝到HDFS中；而-copyFromLocal则更严格限制只能拷贝本地文件到HDFS中。</p>
<p><br></p>
<p>？？？</p>
<p>PS：“<span style="font-size:13px;line-height:16.899999618530273px;"> put would prefer the HDFS scheme instead of the local file system</span>”，也就是说，如果本地和HDFS上都存在相同路径，则-put跟趋于优先取HDFS的源。</p>
<p>但是我测试了：</p>
<p>hadoop fs -put hdfs:///tmp/hive-XXX/test.txt /user/XXX/test.txt.hdfs<br></p>
<p>hadoop fs -put /tmp/hive-XXX/test.txt /user/XXX/test.txt.local       <br></p>
<p>hadoop fs -cat /user/XXX/test.txt.*    <br></p>
<p><br></p>
<p>local path:/tmp/hive-XXX<br>
local path:/tmp/hive-XXX<br></p>
<p><br></p>
<p>所以。。。。</p>
<p><br></p>

Let's make an example: If your HDFS contains the path: <code style="border:0px;font-size:13px;font-family:Consolas, Menlo, Monaco, 'Lucida Console', 'Liberation Mono', 'DejaVu Sans Mono', 'Bitstream Vera Sans Mono', 'Courier New', monospace, sans-serif;background-color:rgb(238,238,238);">/tmp/dir/abc.txt</code> And
 if your local disk also contains this path then the hdfs API won't know which one you mean, unless you specify a scheme like <code style="border:0px;font-size:13px;font-family:Consolas, Menlo, Monaco, 'Lucida Console', 'Liberation Mono', 'DejaVu Sans Mono', 'Bitstream Vera Sans Mono', 'Courier New', monospace, sans-serif;background-color:rgb(238,238,238);">file://</code> or <code style="border:0px;font-size:13px;font-family:Consolas, Menlo, Monaco, 'Lucida Console', 'Liberation Mono', 'DejaVu Sans Mono', 'Bitstream Vera Sans Mono', 'Courier New', monospace, sans-serif;background-color:rgb(238,238,238);">hdfs://</code>.
 Maybe it picks the path you did not want to copy.</p>
<p style="border:0px;clear:both;">
Therefore you have <code style="border:0px;font-size:13px;font-family:Consolas, Menlo, Monaco, 'Lucida Console', 'Liberation Mono', 'DejaVu Sans Mono', 'Bitstream Vera Sans Mono', 'Courier New', monospace, sans-serif;background-color:rgb(238,238,238);">-copyFromLocal</code> which
 is preventing you from accidentally copying the wrong file, by limiting the parameter you give to the local filesystem.</p>
<p style="border:0px;clear:both;">
<code style="border:0px;font-size:13px;font-family:Consolas, Menlo, Monaco, 'Lucida Console', 'Liberation Mono', 'DejaVu Sans Mono', 'Bitstream Vera Sans Mono', 'Courier New', monospace, sans-serif;background-color:rgb(238,238,238);">Put</code> is
 for more advanced users who know which scheme to put in front.</p>
<p style="border:0px;clear:both;">
It is always a bit confusing to new Hadoop users which filesystem they are currently in and where their files actually are.</p>
